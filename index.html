SKILL.md 技能，挂到步骤上会作为该步的系统提示。
占位符：{{input}} 初始输入 · {{prev}} 上一步输出 · {{1}}{{2}}… 第 N 步输出
参数池（可选，注入文章的变量）
地点卡片（可选，可拖拽排序）
输出：带 H1–H4 层级的结构文 + 一条小推文（同次生成）。每个标题段都自成一段、能单独读懂。右侧结果可直接改。
参与模型（按接力顺序，可拖拽排序）
接力改进：第 1 位起草，后面每位在上一位的答案上改进；跑完所有轮后，主持人综述「共识 / 分歧 / 最终建议」。右侧每张结果都能直接改。
参与 Agent（仅本地 CLI，按接力顺序，可拖拽）
点「运行」→ 切到终端，每个 Agent 各开一个实时交互终端、并排在同一文件夹里干活（你能实时看、随时打字让它改）。 队友通过项目根的 TEAM_NOTES.md 互通反馈。勾了「自动批准」就无人值守 （Claude --dangerously-skip-permissions、Codex --dangerously-bypass-approvals-and-sandbox、Gemini --yolo、Grok --always-approve、Cursor --force）—— ⚠️ 它会不经询问改文件 / 跑命令，务必先用 git 管好该项目以便回滚。
单模型多轮对话；CLI 与 API 都支持。每轮会把之前的对话作为上下文一起发给模型。
⚠️ 此来源是终端 CLI，只出 SVG 线条图、不是真实照片。要真实照片请选 📷 开头的来源（图片模型，需先在「接入」填 API Key）。
文生图：直接写描述出图。图生图：再放一张参考图，在它基础上改。终端 CLI（codex / grok 等）出的是 SVG 矢量图；HTTP 图片源（如火山 Seedream）出真实位图、且支持图生图。
