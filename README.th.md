## มันคืออะไร

`council` ให้คุณเชื่อมโยงโมเดลหลายตัวเข้าเป็น **ไปป์ไลน์** แบบเส้นตรง แต่ละขั้นเลือกโมเดลของตัวเอง
(API ที่โฮสต์ไว้ หรือ CLI ในเครื่องอย่าง Claude Code / Codex) และทำงานเพียงอย่างเดียว — ร่าง แก้ไข
ตรวจสอบข้อเท็จจริง ขัดเกลา — แล้วส่งผลลัพธ์ต่อไปยังขั้นถัดไป นอกจากนี้ยังมีโหมด **บทความเดี่ยว**
แบบครั้งเดียวจบ (GEO) สำหรับสร้างบทความที่เสร็จสมบูรณ์ + โพสต์โซเชียลในรอบเดียว พร้อมตัวเลือก
การสร้างภาพแบบอินไลน์

ทุกอย่างทำงานในเครื่องในรูปแบบแอปเดสก์ท็อปเนทีฟ **คีย์ API อยู่เฉพาะใน localStorage ของเบราว์เซอร์
ของคุณเท่านั้น** — ไม่เคยถูกส่งไปที่ใดเลยนอกจาก endpoint ของผู้ให้บริการที่คุณตั้งค่าไว้

## คุณสมบัติ

- **ไปป์ไลน์ข้อความ** — หลายขั้น แต่ละขั้นมีโมเดลและคำสั่งของตัวเอง อ้างอิงผลลัพธ์จากขั้นก่อนหน้า
  ได้ด้วยตัวยึดตำแหน่ง: `{{input}}` (อินพุตเริ่มต้น), `{{prev}}` (ขั้นก่อนหน้า), `{{1}}` `{{2}}` … (ขั้นที่ N)
- **ตลาดโมเดล** — มีพรีเซ็ตสำหรับ DeepSeek, OpenAI, Gemini, Qwen, Kimi, GLM, xAI Grok, Mistral,
  Volcengine Doubao, Ollama, MiniMax endpoint `/chat/completions` ที่เข้ากันได้กับ OpenAI ใดก็ตาม
  ใช้งานได้ด้วยการเพิ่มผู้ให้บริการ
- **เวิร์กเกอร์ CLI ในเครื่อง** — ขับเคลื่อนเอเจนต์ในเครื่อง (Claude Code, Codex, Gemini CLI, Grok CLI)
  ให้เป็นขั้นในไปป์ไลน์ผ่าน `cli_run` แบบทั่วไป (โปรแกรม + อาร์กิวเมนต์ + พรอมป์ต)
- **คลังทักษะ** — พรอมป์ต `SKILL.md` ที่นำกลับมาใช้ซ้ำได้ใน `~/.council/skills` แนบได้ต่อขั้น
  นำเข้าจากไฟล์/โฟลเดอร์ในเครื่อง หรือซิงก์กับ git repo (ดาวน์โหลด / อัปโหลด)
- **โหมดบทความเดี่ยว (GEO)** — ตัวสร้างแบบสแตนด์อะโลน: ชื่อเรื่อง/หัวข้อ, เส้นทาง/สถานที่ (ไม่บังคับ),
  สไตล์การเขียน 10 แบบ, แถบเลื่อนความยาว, ป้อนวัตถุดิบหรือ URL อ้างอิง, ภาพแบบอินไลน์ (ไม่บังคับ)
  ให้ผลเป็นบทความที่แก้ไขได้ + โพสต์โซเชียลสั้น ๆ คัดลอกหรือส่งออกเป็น Markdown ได้
- **การสร้างภาพ** — แปลงข้อความเป็นภาพผ่าน endpoint สไตล์ OpenAI-images (เช่น Volcengine Seedream)
  หรือ SVG ผ่านเวิร์กเกอร์ CLI ในเครื่อง
- **การสร้างวิดีโอ** — แปลงข้อความเป็นวิดีโอแบบ async (Volcengine Ark / Seedance) แสดงผลในการ์ดผลลัพธ์
- **เวิร์กโฟลว์ที่ตั้งชื่อได้** — บันทึก / โหลด / ลบ ไปป์ไลน์ทั้งชุดเป็นไฟล์

## วิธีรัน

ข้อกำหนด: [Node.js](https://nodejs.org/) 18+, [Rust](https://rustup.rs/) (เวอร์ชัน stable) และ
[ข้อกำหนดเบื้องต้นของ Tauri 2](https://tauri.app/start/prerequisites/) สำหรับระบบปฏิบัติการของคุณ

```sh
npm install
npm run tauri dev      # development mode, opens a window with HMR
npm run tauri build    # build a release .app / .dmg (or platform equivalent)
```

## การใช้งาน

1. เปิด **厂商 / 命令 / Key** (ผู้ให้บริการ / คำสั่ง / คีย์) ในแถบด้านบนแล้ววางคีย์ API ของคุณ
   มีผู้ให้บริการ DeepSeek ตั้งไว้ให้แล้ว — แค่เพิ่มคีย์ สำหรับบริการอื่นที่เข้ากันได้กับ OpenAI
   คลิกเพื่อเพิ่มผู้ให้บริการแล้วตั้งค่า Endpoint จนถึงและรวมถึง `/chat/completions`
2. ในแผงด้านซ้าย สร้างไปป์ไลน์: แต่ละขั้นเลือกโมเดล (หรือ CLI ในเครื่อง) และคำสั่ง
3. ใช้ตัวยึดตำแหน่งในคำสั่งเพื่ออ้างอิงผลลัพธ์จากขั้นก่อนหน้า:
   - `{{input}}` — อินพุตเริ่มต้นที่ด้านบน
   - `{{prev}}` — ผลลัพธ์ของขั้นก่อนหน้า
   - `{{1}}` `{{2}}` … — ผลลัพธ์ของขั้นที่ N
4. คลิก **▶ 运行 (รัน)** ขั้นต่าง ๆ จะทำงานจากบนลงล่างและสตรีมเข้าสู่แผงผลลัพธ์ทางด้านขวา
5. สลับไปที่ **单篇 (บทความเดี่ยว)** ในแถบด้านบนเพื่อใช้ตัวสร้าง GEO แบบครั้งเดียวจบ

## สถาปัตยกรรม

แบ็กเอนด์ Rust (`src-tauri/src/lib.rs`) เปิดเผยคำสั่ง Tauri จำนวนหนึ่ง ส่วนหน้าบ้านที่เป็น
vanilla-TS ทำหน้าที่ประสานงานไปป์ไลน์และสตรีมแต่ละขั้น

| Command | จุดประสงค์ |
| --- | --- |
| `chat_stream` | `/chat/completions` ที่เข้ากันได้กับ OpenAI (SSE) สตรีม delta ผ่าน Tauri Channel |
| `cli_run` | รันเวิร์กเกอร์ CLI ในเครื่อง (โปรแกรม + อาร์กิวเมนต์ที่กำหนดไว้ + พรอมป์ตเป็น argv สุดท้าย) |
| `fetch_url` | ดึงหน้าเว็บและสกัดข้อความที่อ่านได้ (สำหรับการป้อน URL อ้างอิง) |
| `image_generate` | แปลงข้อความเป็นภาพ (สไตล์ OpenAI-images คืนค่า URL ของภาพ) |
| `video_generate` | API งานแปลงข้อความเป็นวิดีโอแบบ async (ส่ง + โพล) คืนค่า URL ของวิดีโอ |
| `*_workflow` / `*_skill` | บันทึก / โหลด / แสดงรายการ / ลบ เวิร์กโฟลว์และทักษะ; ดาวน์โหลด / อัปโหลดทักษะผ่าน git |

- **การสตรีม**: ทั้งเวิร์กเกอร์ HTTP และ CLI ดันข้อความแบบเพิ่มทีละส่วนไปยังหน้าบ้านผ่าน Tauri
  `Channel<StreamEvent>` หน้าบ้านจัดการพวกมันอย่างเป็นเอกภาพ
- **`reqwest`** ใช้ `rustls-tls` (ไม่ต้องพึ่ง OpenSSL ของระบบ)
- **คีย์** ถูกเก็บไว้ใน localStorage เท่านั้น การหยุดการรันจะทำให้หน้าบ้านหยุดรับฟัง ส่วนคำขอ HTTP
  ของแบ็กเอนด์ที่กำลังดำเนินอยู่จะทำงานจนเสร็จในเบื้องหลัง

## โรดแมป

- การเขียนร่วมแบบหลายโมเดลสำหรับโหมดบทความเดี่ยว (สายงานนักเขียน→บรรณาธิการ / รูปแบบขนานเพื่อเปรียบเทียบ)
