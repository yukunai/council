## यह क्या है

`council` आपको कई मॉडलों को एक रेखीय **पाइपलाइन** में जोड़ने देता है। हर चरण अपना खुद का
मॉडल चुनता है (एक होस्टेड API, या Claude Code / Codex जैसा कोई लोकल CLI) और एक ही काम करता है — मसौदा बनाना, संशोधन,
तथ्य-जाँच, परिष्करण — अपना आउटपुट आगे भेजते हुए। एक वन-शॉट **सिंगल-आर्टिकल**
मोड (GEO) भी है, जो एक ही पास में तैयार लेख + सोशल पोस्ट तैयार करता है, साथ ही वैकल्पिक
इनलाइन इमेज जनरेशन के साथ।

सब कुछ एक नेटिव डेस्कटॉप ऐप के रूप में लोकल चलता है। **API कीज़ केवल आपके ब्राउज़र के
localStorage में रहती हैं** — आप जो प्रोवाइडर एंडपॉइंट कॉन्फ़िगर करते हैं, उसके अलावा वे कभी कहीं नहीं भेजी जातीं।

## विशेषताएँ

- **टेक्स्ट पाइपलाइन** — कई चरण, हर एक का अपना मॉडल और निर्देश। अपस्ट्रीम
  आउटपुट को प्लेसहोल्डर्स के ज़रिए संदर्भित करें: `{{input}}` (शुरुआती इनपुट), `{{prev}}` (पिछला चरण), `{{1}}` `{{2}}` … (चरण N)।
- **मॉडल मार्केट** — DeepSeek, OpenAI, Gemini, Qwen, Kimi, GLM, xAI Grok, Mistral,
  Volcengine Doubao, Ollama, MiniMax के लिए प्रीसेट। कोई भी OpenAI-संगत `/chat/completions` एंडपॉइंट
  एक प्रोवाइडर जोड़कर काम करता है।
- **लोकल CLI वर्कर्स** — एक जेनेरिक `cli_run` (प्रोग्राम + आर्ग्स + प्रॉम्प्ट) के ज़रिए लोकल एजेंट्स
  (Claude Code, Codex, Gemini CLI, Grok CLI) को पाइपलाइन चरणों के रूप में चलाएँ।
- **स्किल्स लाइब्रेरी** — `~/.council/skills` में पुनः-उपयोग योग्य `SKILL.md` प्रॉम्प्ट, प्रति चरण संलग्न किए जा सकते हैं।
  लोकल फ़ाइलों/फ़ोल्डरों से इम्पोर्ट करें, या किसी git रिपॉज़िटरी के साथ सिंक करें (डाउनलोड / अपलोड)।
- **सिंगल-आर्टिकल (GEO) मोड** — एक स्वतंत्र जनरेटर: शीर्षक/विषय, वैकल्पिक रूट/स्थान,
  10 लेखन शैलियाँ, लंबाई स्लाइडर, इसे कच्चा सामग्री या एक संदर्भ URL दें, वैकल्पिक इनलाइन
  इमेज। एक संपादन-योग्य लेख + एक छोटी सोशल पोस्ट देता है; कॉपी करें या Markdown में एक्सपोर्ट करें।
- **इमेज जनरेशन** — OpenAI-images-शैली एंडपॉइंट्स (जैसे Volcengine Seedream) के ज़रिए टेक्स्ट-टू-इमेज,
  या एक लोकल CLI वर्कर के ज़रिए SVG।
- **वीडियो जनरेशन** — एसिंक्रोनस टेक्स्ट-टू-वीडियो (Volcengine Ark / Seedance), रिज़ल्ट कार्ड में रेंडर किया गया।
- **नामित वर्कफ़्लो** — पूरी पाइपलाइनों को फ़ाइलों के रूप में सेव / लोड / डिलीट करें।

## इसे चलाएँ

आवश्यकताएँ: [Node.js](https://nodejs.org/) 18+, [Rust](https://rustup.rs/) (stable), और आपके OS के लिए
[Tauri 2 prerequisites](https://tauri.app/start/prerequisites/)।

```sh
npm install
npm run tauri dev      # development mode, opens a window with HMR
npm run tauri build    # build a release .app / .dmg (or platform equivalent)
```

## उपयोग

1. टॉप बार में **厂商 / 命令 / Key** (Providers / Commands / Keys) खोलें और अपनी API की पेस्ट करें।
   एक DeepSeek प्रोवाइडर पहले से मौजूद है — बस की जोड़ें। किसी भी अन्य OpenAI-संगत सेवा के लिए,
   एक प्रोवाइडर जोड़ने के लिए क्लिक करें और उसका Endpoint `/chat/completions` तक (इसे शामिल करते हुए) सेट करें।
2. बाएँ पैनल में, पाइपलाइन बनाएँ: हर चरण एक मॉडल (या एक लोकल CLI) और एक निर्देश चुनता है।
3. अपस्ट्रीम आउटपुट को संदर्भित करने के लिए निर्देशों में प्लेसहोल्डर्स का उपयोग करें:
   - `{{input}}` — सबसे ऊपर का शुरुआती इनपुट
   - `{{prev}}` — पिछले चरण का आउटपुट
   - `{{1}}` `{{2}}` … — चरण N का आउटपुट
4. **▶ 运行 (Run)** पर क्लिक करें। चरण ऊपर से नीचे तक चलते हैं और दाएँ-हाथ के रिज़ल्ट पैनल में स्ट्रीम होते हैं।
5. वन-शॉट GEO जनरेटर के लिए टॉप बार में **单篇 (Single-article)** पर स्विच करें।

## आर्किटेक्चर

Rust बैकएंड (`src-tauri/src/lib.rs`) कुछ Tauri कमांड्स एक्सपोज़ करता है; vanilla-TS
फ़्रंटएंड पाइपलाइन को ऑर्केस्ट्रेट करता है और हर चरण को स्ट्रीम करता है।

| Command | उद्देश्य |
| --- | --- |
| `chat_stream` | OpenAI-संगत `/chat/completions` (SSE), एक Tauri Channel पर डेल्टा स्ट्रीम करता है |
| `cli_run` | एक लोकल CLI वर्कर चलाता है (प्रोग्राम + फ़िक्स्ड आर्ग्स + अंतिम argv के रूप में प्रॉम्प्ट) |
| `fetch_url` | एक वेब पेज लाता है और पठनीय टेक्स्ट निकालता है (संदर्भ-URL फीडिंग के लिए) |
| `image_generate` | टेक्स्ट-टू-इमेज (OpenAI-images-शैली, एक इमेज URL लौटाता है) |
| `video_generate` | एसिंक्रोनस टेक्स्ट-टू-वीडियो टास्क API (सबमिट + पोल), एक वीडियो URL लौटाता है |
| `*_workflow` / `*_skill` | वर्कफ़्लो और स्किल्स को सेव / लोड / लिस्ट / डिलीट करता है; स्किल्स का git डाउनलोड / अपलोड |

- **स्ट्रीमिंग**: HTTP और CLI दोनों वर्कर्स एक Tauri `Channel<StreamEvent>` पर फ़्रंटएंड को इन्क्रीमेंटल
  टेक्स्ट पुश करते हैं; फ़्रंटएंड उन्हें एकसमान रूप से संभालता है।
- **`reqwest`** `rustls-tls` का उपयोग करता है (कोई सिस्टम OpenSSL निर्भरता नहीं)।
- **कीज़** केवल localStorage में संग्रहीत होती हैं। किसी रन को रोकने से फ़्रंटएंड सुनना बंद कर देता है; एक
  इन-फ़्लाइट बैकएंड HTTP रिक्वेस्ट बैकग्राउंड में पूरी हो जाती है।

## रोडमैप

- सिंगल-आर्टिकल मोड के लिए मल्टी-मॉडल सह-लेखन (लेखक→संपादक श्रृंखला / तुलना के लिए समानांतर वैरिएंट)।
- राउंड-टेबल चर्चा मोड (एक ही प्रश्न, कई मॉडल, कई राउंड + सारांश)।
